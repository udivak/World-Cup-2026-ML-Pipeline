Executive summary

The system predicts World Cup matches bottom-up — team strength is computed from the players actually in each squad (FIFA/EA-FC ratings), not from team identity. Phases 0–2 are complete: a clean data pipeline feeds 382 team profiles built from real tournament rosters. Phase 3 turns those profiles into per-match features and trains calibrated Win/Draw/Loss models, then judges them with a leakage-free across-edition backtest.

The success gate — beat both reference baselines (Elo and squad-overall) on RPS and log-loss — did not pass. The FIFA-only profile model is competitive (RPS ≈ 0.19, near bookmaker-grade) and the bottom-up signal is genuinely present, but a classic Elo baseline still predicts better on every metric. This is a real, honestly-reported result and a natural decision checkpoint — the data the design intended (Football Manager attributes, caps / experience, recent form) is not yet in the profiles.

The model layer (Phase 3)

Features (single train/serve code path). For each match, 13 team1−team2 profile differences (unit strengths GK/DEF/MID/ATT, overall XI, star power, depth, caps, age, market value, top-5-league share, composure) plus context: home_adv (0 at neutral venues) and cross_conf. The realized result is the label — never a feature.

Two reference baselines bar to beat a chronological football-Elo (rolled over the full 49.4k-match history → 3-way logit) and a naive squad-overall logit on a single feature. Two contenders profile multinomial LogReg and HistGradientBoosting, plus their ensemble. Everything is probability-calibrated.

Validation: expanding window across editions (train on all finals before a tournament, test on it, roll forward) — never a random split. No-leakage is enforced by tests: every squad/attribute feeding a match predates it (a 2018 World Cup squad uses FIFA 18, not 19).

Why the gate failed

  • Profiles are FIFA-only and thin. The experience / mentality columns the design called for — total_wc_apps, total_continental_apps, mean_work_rate — are still NULL (Football Manager attributes and real caps were never ingested; they degrade to null).
  • Low confederation coverage. Gold Cup / Asian Cup / AFCON squads match only ~11–16 of 26 players to FIFA, so their unit-strength diffs are noisy — exactly where Elo wins biggest.
  • No recent-form signal. Elo absorbs results up to the week before kickoff; the profile uses a FIFA snapshot up to ~8 months old.
  • Adding profile richness beyond overall XI hurt RPS — the naive squad-overall baseline (0.1867) outscored the full profile models, confirming the extra features are mostly noise under current coverage.

Next steps

Enrich data, then retry recommended
Fill the richness the design intended but the pipeline still lacks: ingest Football Manager attributes and real caps / international experience (the total_wc_apps, total_continental_apps and mean_work_rate profile columns are still NULL), add a recent-form proxy, and lift confederation FIFA coverage (Gold Cup / Asian Cup squads match only ~11–16 of 26 players). Then re-run this exact backtest. Most aligned with the bottom-up thesis; largest effort (reopens Phase 1 data work).
Proceed to Phase 4 as-is
Accept the profile model — RPS ≈ 0.19, near bookmaker-grade and well-calibrated — as the WC2026 predictor despite not beating Elo, with the limitation documented. Gets you 104-fixture predictions and Monte-Carlo tournament odds now; revisit data richness later.
Allow an Elo + profile blend
Relax the “Elo is never a model feature” rule to build the most accurate WC2026 predictor by blending Elo with the profile signal. Likely the best raw accuracy, but deviates from the pure bottom-up design and dilutes the research question.
Merge what's built, decide later
Treat Phase 3 as a complete, honestly-reported build (gate = FAIL recorded) and open the PR for phase-3-model-backtest. Pick a direction in a later session.
